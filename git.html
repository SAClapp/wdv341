Terms Defined:

  • Version Control Software: is used as a way for users to track changes to
    source code and observe all changes made to source code over time.
  • Add: the command $git add filename.ext tells the git version control software or vcs that
    a file is important and gets it ready to commit the file. For example if I wanted to add this page to my git I
    would say $git add git.html and it would be added to the list of files that I want to have ready for my next commit.
  • Commit: the command $git commit -m "insert message here" tells git that the committed files
    are ready to be backed up to your remote repository. When doing this you should also put a message in quotes after the
    -m to help keep track of which version of your file or files is being committed.
    For example I would do $git commit -m "First version of git.html".
  • Push: the command $git push origin master moves committed files to your remote repository.
  • Pull: the command $git pull updates parts your local repository with changes made on your remote
    repository. In other words it helps you keep your local repositories current with updates you or others may have made to files
    on your remote repositories.
  • Clone: the command $git clone creates a copy of an entire repository within a repository.
